# Session Data: Availability, Format, and Scoring Scope

## Quick Summary

CodeFluent scores your AI fluency based on Claude Code session transcripts stored as JSONL files at `~/.claude/projects/`. **Session transcripts are only available from late January 2026 onward** — Claude Code did not persist full session transcripts before that. Subagent sessions (spawned by Claude's Agent tool) are excluded from scoring because they contain AI-generated prompts, not human input.

## Session Data Availability

Claude Code introduced per-session JSONL transcript files around **late January 2026**. Before that, Claude Code stored only prompt summaries and aggregate statistics — not the full conversation transcripts needed for fluency scoring.

If you've been using Claude Code since before late January 2026, you may notice that CodeFluent only shows sessions from that date onward. **This is expected.** The older session data was never saved in a format that supports fluency analysis.

### What Data Exists from Before Late January 2026

| Source | Location | Content | Usable for Scoring? |
|--------|----------|---------|---------------------|
| Prompt history | `~/.claude/history.jsonl` | One-line prompt summaries, timestamps, project path | No — lacks assistant responses, tool use details, and full prompt text |
| Usage stats | `~/.claude/stats-cache.json` | Aggregate daily counts (sessions, messages, tool calls) | No — aggregate counts only, no prompt text |
| File history | `~/.claude/file-history/` | File snapshots versioned per session | No — file contents only, not conversations |
| Debug logs | `~/.claude/debug/` | Debug output per session | No — debug data, not session transcripts |
| MCP logs | `~/.cache/claude-cli-nodejs/` | MCP tool communication logs | No — tool protocol logs, not session data |

### Why Some Projects May Be Missing

Projects used exclusively before the JSONL format was introduced may have no directory at all under `~/.claude/projects/`. This is expected — those sessions were never persisted as transcripts. The `~/.claude/history.jsonl` file is the only record that those sessions occurred.

## Current Session Storage Format

Claude Code stores session data at `~/.claude/projects/` using the following structure:

```
~/.claude/projects/
├── <project-path-encoded>/                    # One directory per project
│   ├── <session-uuid>.jsonl                   # Main session transcript (flat file)
│   ├── <session-uuid>/                        # Optional support directory
│   │   ├── subagents/                         # AI-spawned subagent sessions
│   │   │   └── agent-<id>.jsonl               # Subagent transcript
│   │   └── tool-results/                      # Tool output artifacts
│   │       └── toolu_<id>.txt
│   └── memory/                                # Project memory (not a session)
│       └── MEMORY.md
```

### Key details

- **Main sessions** are flat `.jsonl` files named with a UUID (e.g., `1a67a55f-b4ed-45f9-8347-bde22cd0609c.jsonl`)
- **Support directories** share the same UUID as the flat file and contain subagent data and tool results
- **Project path encoding** replaces `/` with `-` in the project path (e.g., `/home/user/my-project` becomes `-home-user-my-project`)
- Both the flat file and its support directory may coexist for the same session
- Some support directories exist without a corresponding flat file — these contain only subagent and tool-result data

## What Gets Scored (and What Doesn't)

### Included in fluency scoring

- **Main session prompts** — your typed prompts from the flat `.jsonl` files
- **Session metadata** — plan mode usage, tool diversity, thinking count, model, timestamps
- **CLAUDE.md config** — your project's CLAUDE.md file is scored separately for fluency behaviors and merged via `session OR config` logic

### Excluded from fluency scoring

- **Subagent sessions** — JSONL files with `isSidechain: true` are skipped. These contain prompts generated by Claude's Agent tool, not your human input. Scoring them would inflate session counts and misrepresent your actual fluency.
- **Tool results** — raw output from tool executions (stored in `tool-results/` directories)
- **System messages, progress events, file snapshots** — metadata types that don't represent user interaction
- **Interrupted prompts** — messages containing only `[Request interrupted by user for tool use]`

### How to verify your session count

The number of sessions shown in CodeFluent's dropdown should match the number of `.jsonl` files in your `~/.claude/projects/` directories (excluding subagent files). You can check with:

```bash
# Count main session files (flat .jsonl files only)
find ~/.claude/projects -maxdepth 2 -name "*.jsonl" | wc -l
```

## JSONL Session Format

Each session file contains one JSON object per line. The key message types used for scoring:

| Type | Contains | Used for |
|------|----------|----------|
| `user` | User prompts (`message.content`), plan mode flag (`planContent`) | Fluency behavior analysis |
| `assistant` | Claude responses, model info, tool use blocks | Tool diversity counting |
| `tool_use` | Top-level tool invocations | Tool usage statistics |
| `thinking` | Extended thinking events | Thinking count signal |

User prompt content can be either a plain string or an array of content blocks (e.g., `[{"type": "text", "text": "..."}]`). The parser handles both formats.
